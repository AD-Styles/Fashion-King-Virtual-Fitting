# 계획서 작성용 Claude 프롬프트 모음

학교 과제 / 졸업작품 / 캡스톤 디자인 / 팀프로젝트 계획서를 Claude로 작성하기 위한 프롬프트 세트.

## 사용법

1. **Claude.ai** 또는 Claude Code에서 사용. Sonnet/Opus 모두 가능 (긴 문서는 Opus 권장).
2. 아래 `[대괄호]` 부분을 본인 상황에 맞게 **반드시 채워서** 던지기.
3. 순서대로 진행: 메인 → 보조 1 → 보조 2 → ...
4. 한 번의 응답에서 출력이 잘리면 "이어서 작성해주세요"라고 답.

<project>
**프로젝트명**: 내일은패션왕 (Naeil Fashion King)

**한 줄 요약**: 가상 마네킹에 옷을 입혀보고 360도 돌려보는 웹 기반 패션 시뮬레이터

**핵심 컨셉**:
사용자가 자신의 체형을 닮은 가상 마네킹을 만들고, 다양한 옷을 입혀본 뒤 360도 회전하며 모든 각도에서 코디를 미리 확인할 수 있는 웹 애플리케이션. 추후 AI 기술을 통합하여 옷 사진 한 장으로 가상 옷을 생성하고, 사용자 사진 기반의 개인화된 마네킹을 제공하는 방향으로 확장.

**해결하려는 문제**:
1. 온라인 쇼핑에서 옷을 사기 전에 핏을 미리 보기 어려움 (반품률 高)
2. 다양한 코디 조합을 시도해보는 비용(시간·공간)이 큼
3. 연예인이나 인플루언서의 룩을 따라 입어보고 싶을 때 사전 시뮬레이션 불가

**타깃 사용자**:
- 10~30대 패션에 관심 많은 사용자
- 온라인 쇼핑 헤비 유저
- 자신의 체형과 어울리는 옷을 찾고 싶은 사용자

**기술 스택**:
- 프론트엔드: Next.js 14 (App Router), TypeScript, Tailwind CSS
- 3D 렌더링: Three.js, React Three Fiber, Drei
- 상태 관리: Zustand
- AI 모델 (Phase 3+): TripoSR / Wonder3D (3D 메쉬 생성), IDM-VTON (2D 합성), Stable Diffusion (텍스처 생성) 후보군
- 플랫폼: 웹 우선, 모바일은 추후 확장

**개발 단계 (Phase)**:
- Phase 1 (현재): 체형 커스터마이징 — 키·몸무게·어깨·가슴·허리·엉덩이·다리 등 7개 파라미터 슬라이더로 마네킹 조정
- Phase 2: 옷 입히기 — .glb 옷 모델을 마네킹 본(bone)에 부착, 카테고리별 갈아입히기
- Phase 3: AI 옷 생성 — 옷 사진 → 3D 메쉬 또는 텍스처 자동 생성
- Phase 4: 사용자 사진 → 마네킹 개인화

**현재 진행 상황**:
- 프로젝트 초기 셋업 완료 (Next.js + React Three Fiber)
- 임시 프리미티브 마네킹 구현 (Cylinder/Box/Sphere 조합)
- 체형 조정 슬라이더 7개 실시간 작동
- 360° 회전 기능 작동
- 다음 단계: 실제 .glb 마네킹 모델 통합
</project>

## 보조 프롬프트 4 — 데모 시연 시나리오

> 발표 자리에서 실제 시연할 흐름을 짜기. 시연 사고 방지용.

```
앞서 작성한 계획서의 "주요 기능"을 발표 자리에서 실제로 보여줄 데모 시연 시나리오를 작성해 주십시오.

조건:
- 시연 시간: [예: 3분]
- 시연 환경: 노트북 + 프로젝터/대형 모니터, 인터넷 가능 (단, 불안정할 수 있음)
- 시연 페이즈: Phase 1 (체형 커스터마이징) + Phase 2 일부 (가능하다면 옷 1~2벌)

다음을 포함해 주십시오.

1. **시연 흐름 (단계별)**:
   - 단계 N — 화면에서 무엇이 보이는지 / 발표자가 무엇을 클릭/조작하는지 / 구두로 어떤 멘트를 하는지
   - 각 단계는 30초 이내가 되도록 짧게 분할

2. **백업 플랜**:
   - 인터넷 오류 시: 사전 녹화 영상으로 전환 절차
   - 3D 렌더링 오류 시: 스크린샷 슬라이드로 대체
   - 노트북 화면 미러링 실패 시: 모바일 핫스팟 등 대안

3. **강조 포인트**:
   - 청중이 절대 놓치지 말아야 할 순간 3가지 (예: 슬라이더 움직일 때 실시간 반응, 360° 회전 시 부드러움)
   - 각 순간을 발표자가 어떻게 어필할지

4. **사전 준비 체크리스트** (시연 직전 5분):
   - 브라우저 캐시, 환경 변수, 네트워크, 백업 파일 등

5. **실패 시 회복 멘트**:
   - 데모가 깨졌을 때 청중을 잃지 않을 자연스러운 한국어 멘트 2~3개
```

## 사용 팁

- **메인 프롬프트만으로도 80% 완성도**의 계획서가 나옴. 보조는 필요할 때만.
- 받은 결과 중 마음에 안 드는 섹션은 "X 섹션을 더 [구체적으로 / 짧게 / 학술적으로] 다시 써주세요"로 부분 재요청.
- 같은 대화 안에서 후속 프롬프트를 던질 것 (컨텍스트 유지). 다른 대화로 옮기면 프로젝트 정보를 다시 다 붙여야 함.
- Claude.ai Project 기능을 쓰면 프로젝트 정보를 한 번 등록하고 여러 대화에서 재사용 가능.
- 출력 길이가 부족하면 "이어서 작성해주세요"로 계속.
- 사실 검증(특히 통계와 논문 인용)은 반드시 사람이 한 번 확인. Claude가 가짜 수치를 만들 수 있음.
