## 모듈별 구현 상세

### 모듈 1: 퍼스널컬러 CNN 학습 파이프라인

#### 1.1 학습 데이터 준비 (label_dataset.py)

**목표:** FairFace HuggingFace 코퍼스에서 얼굴 이미지를 크롭하고, 피부/머리/눈 색을 자동 추출하여 의사라벨을 생성한다.

**구현:**

- **Pass 1 (얼굴 추출 및 색 추출)**
  - FairFace 데이터셋에서 이미지 다운로드
  - mediapipe FaceLandmarker로 얼굴 감지
  - features.py의 `extract_face_colors(method=seg)` 호출:
    - SegFormer 페이스 파싱으로 피부/머리/눈 마스크 생성
    - 또는 method=landmark로 mediapipe 랜드마크 주변 패치 샘플링
  - 추출된 색 RGB를 representative_rgb() 함수로 정제
    - Lab L* 극단값 drop_frac(기본 0.2) 비율 제거 (이상치 거부)
    - 중앙값 계산으로 robust mean 확보
  - 224×224 JPG로 저장

- **Pass 2 (규칙 기반 의사라벨 + 중립점 보정)**
  - 수집된 모든 이미지에 대해 피부/머리 색 중앙값 계산
  - rules.compute_axes() 함수의 중립점(B_NEUTRAL, L_NEUTRAL 등) 을 코퍼스 중앙값으로 재설정
  - 각 이미지의 색 -> 3축(warmcool/value/clarity) -> 12 타입 분류
  - labels.csv 출력: filename, warmcool, value, clarity

**검증:** features_selftest.py로 representative_rgb의 이상치 거부 검증 (피부색 + 반사광 + 그림자 시나리오)

#### 1.2 CNN 학습 (train.py)

**모델 아키텍처:**
- ResNet18 (ImageNet 사전학습)
- 마지막 FC 계층을 512 -> 3으로 변경 (warmcool/value/clarity 출력)

**학습 설정:**
- 손실함수: SmoothL1Loss (회귀용, 이상치에 로버스트)
- 옵티마이저: AdamW, lr=3e-4, weight_decay=1e-4
- Batch size: 32
- Epochs: 20
- 데이터 증강: 기하 증강만 (ColorJitter 제외 - 색 신호 오염 방지)
  - RandomResizedCrop, RandomHorizontalFlip, RandomRotation
- 정규화: ImageNet 평균/표준편차 (mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])

**검증 메트릭 (evaluate 함수):**
- 축 평균절대오차(MAE): warmcool, value, clarity
- 파생 시즌 정확도: derive_type()으로 회귀값 -> 시즌 변환 후 정확도 계산
- 파생 타입 정확도: 12 타입 분류 정확도

**최고 성능:**
- val_season_acc = 90.375% (0.90375)
- val_type_acc = 79.5% (0.795)
- val_mae: warmcool=0.1776, value=0.1371, clarity=0.2683

**저장:** 최고 season_acc일 때의 모델을 personal_color_resnet18.pt로 저장
- 사이드카 JSON 포함: img_size, normalize, axes, val_mae, val_season_acc, val_type_acc, epoch

#### 1.3 색 분류 규칙 (rules.py, taxonomy.py)

**규칙 설계:**

```
3축 (warmcool, value, clarity) -> 4시즌 2단계 분류
```

**Step 1: 4시즌 결정**
```
warmcool >= 0.0 and value < 0.0  → 봄 (spring, 웜+라이트)
warmcool < 0.0 and value < 0.0   → 여름 (summer, 쿨+라이트)
warmcool >= 0.0 and value >= 0.0 → 가을 (autumn, 웜+딥)
warmcool < 0.0 and value >= 0.0  → 겨울 (winter, 쿨+딥)
```

**Step 2: 세부 타입 결정 (STRONG=0.6 임계값)**
```
봄: clarity >= 0.6 → bright, value <= -0.6 → light, 기타 → true
여름: clarity <= -0.6 → soft, value <= -0.6 → light, 기타 → true
가을: clarity <= -0.6 → soft, value >= 0.6 → deep, 기타 → true
겨울: clarity >= 0.6 → bright, value >= 0.6 → deep, 기타 → true
```

**12개 타입별 팔레트:**
- 각 타입마다 hex 7색 배열 (의류 추천용)
- 예: spring_light = ["#F8D7BE", "#FFE0AC", ..., "#F6E7CE"]
- 예: winter_true = ["#0046B8", "#C2185B", ..., "#00838F"]

**신뢰도 계산:**
```
confidence = tanh(min(|warmcool|, |value|))
```
- 두 주축(warmcool, value)이 명확할수록 신뢰도 증가
- 경계선 케이스(두 축 모두 약함) -> 낮은 신뢰도

#### 1.4 축 계산 (rules.compute_axes)

```python
def compute_axes(f):
    skin = srgb_to_lab(f.skin_rgb)  # CIELab 변환
    hair = srgb_to_lab(f.hair_rgb)
    
    # 웜-쿨: 피부 황색도(b*) + 머리 보조
    warmcool = (skin_b - B_NEUTRAL) / 10.0 + 0.3 * (hair_b / 10.0)
    
    # 밝기-깊이: 피부 L*
    value = (L_NEUTRAL - skin_L) / 15.0
    
    # 선명-탁함: 피부-머리 대비 + 피부 채도
    contrast = abs(skin_L - hair_L)
    clarity = (contrast - CONTRAST_NEUTRAL) / 20.0 + (chroma(skin) - CHROMA_NEUTRAL) / 20.0
```

**중립점 (초기값 - label_dataset.py가 재보정):**
- B_NEUTRAL = 14.0 (피부 b* 중립)
- L_NEUTRAL = 62.0 (피부 L* 중립)
- CONTRAST_NEUTRAL = 35.0 (피부-머리 L* 대비 중립)
- CHROMA_NEUTRAL = 18.0 (피부 채도 중립)

### 모듈 3: 색조화도 계산 및 의류 추천 (프론트엔드)

#### 3.1 색공간 변환 (colorHarmony.ts)

**sRGB ↔ CIELab 변환:**
- sRGB(0~255) -> linear RGB -> XYZ -> CIELab (D65)
- D65 백색점: (X,Y,Z) 정규화 (Xn=0.95047, Yn=1.0, Zn=1.08883)
- 역함수: sRGB = f(t > 0.008856 ? t^(1/3) : 7.787*t + 16/116)
- 출력: Lab = [L(0~100), a(-127~127), b(-127~127)]

**ΔE76 거리 계산:**
```javascript
function deltaE76(a: Lab, b: Lab): number {
  const dl = a[0] - b[0];  // L* 거리
  const da = a[1] - b[1];  // a* 거리
  const db = a[2] - b[2];  // b* 거리
  return Math.sqrt(dl*dl + da*da + db*db);
}
```

#### 3.2 의류 추천 필터링 (colorHarmony.ts)

**rankOutfitsByPalette 함수:**
1. 팔레트(7색 hex) -> Lab 변환
2. 각 등록 옷의 색 -> Lab 변환
3. 각 옷을 가장 가까운 팔레트 색까지의 ΔE 계산
4. ΔE 오름차순으로 정렬

**harmonyLabel 함수:**
```javascript
if (deltaE < 12)  -> "아주 잘 어울림" (good)
if (deltaE < 28)  -> "무난함" (ok)
else              -> "덜 어울림" (bad)
```

**UI 통합 (PersonalColorPanel.tsx):**
- 시즌 팔레트 시각화 (7개 색칩)
- 신뢰도 표시 (confidence 프로그레스 바)
- 3축 값 시각화 (warmcool/value/clarity 수평 바)
- 카테고리별 추천 옷 필터링
  - "아주 잘 어울림" (ΔE<12): 우선 추천
  - "무난함" (12≤ΔE<28): 추천
  - "덜 어울림" (ΔE≥28): missingCats에 분리 표시

## 기술적 설계 결정

### 1. 색공간 선택: RGB → CIELab

**의사결정:**
- RGB: 지각적으로 비균일 (유클리드 거리가 인간 감각과 불일치)
- HSV: 색상 조화는 좋으나, 밝기/채도 변화에 민감
- CIELab: 지각적 균일성(perceptual uniformity) 우수

**적용:**
- color_utils.py에서 sRGB ↔ Lab 변환 제공
- colorHarmony.ts에서 동일한 변환 재구현 (프론트엔드 독립성)
- ΔE76 거리로 의류 추천 필터링

### 2. 색 증강 제외 (기하 증강만)

**의사결정:**
- ColorJitter, RandomSaturation 등 색 증강은 라벨 오염
- 라벨은 원본 색(화이트밸런스 미적용)에서 추출
- 따라서 학습 데이터도 원본 색 유지

**적용:**
```python
train_tf = T.Compose([
    T.RandomResizedCrop(size, scale=(0.85, 1.0), ratio=(0.9, 1.1)),
    T.RandomHorizontalFlip(),
    T.RandomRotation(8),
    # ColorJitter 없음
    T.ToTensor(),
    norm,
])
```

### 3. 2-pass 부트스트랩 라벨링

**의사결정:**
- Pass 1: 다양한 피부색 수집 (FairFace 전체 스캔)
- Pass 2: 코퍼스 통계로 중립점 재보정

**이유:**
- 동양인/유럽인/아프리카인 피부색 편차 (특히 황색도 b*)
- 단일 규칙 적용 시 편향된 라벨

**적용:**
```python
# Pass 2에서
B_NEUTRAL = skin_b_corpus_median  # 14.0 -> 실제값으로 재설정
L_NEUTRAL = skin_L_corpus_median  # 62.0 -> 실제값으로 재설정
# ... 외 3개
```

### 4. 학습-추론 분포 일치

**의사결정:**
- 얼굴 크롭 로직을 predict.py와 personal_color.py에서 동일하게 구현

**구현:**
```python
# 학습: FairFace 224×224 정사각 크롭
# 추론: mediapipe bbox * pad(1.25) -> 정사각 -> 224 리사이즈
```

**영향:**
- 학습-추론 분포 일치 -> 정확도 보장
- 불일치 -> 추론 성능 급락

### 5. ONNX 배포 (torch 불필요)

**의사결정:**
- 학습: torch (무거움, 학습 용)
- 추론: onnxruntime (경량, 배포 용)

**이점:**

## 한계 및 향후 과제

### 기술적 한계

1. **규칙 중립점 일반화 부족**
   - 현재: FairFace(동양인 기준 근사치) 데이터셋만 보정
   - 미인정: 아프리카 피부색, 중남미 데이터셋 재보정 미확인
   - 향후: 다양한 인구통계 데이터셋에서 중립점 재보정 실험

2. **얼굴 미검출 시 폴백의 정확도 하락**
   - 현재: 중앙 정사각 크롭 폴백 (학습 분포와 상이)
   - face_detected 플래그로 사용자에게 알림
   - 향후: 더 나은 폴백 전략 (예: 전신 사진에서 얼굴 추정)

3. **신뢰도 지표의 휴리스틱 성질**
   - 현재: confidence = tanh(min(|warmcool|, |value|))
   - 한계: 경계선 케이스 판별 불정확
   - 향후: 베이지안 불확실도 모델 도입

4. **12 타입 분류의 STRONG=0.6 임계값**
   - 현재: 임의 상수 (경험적 조정)
   - 미검증: 코퍼스 통계 기반 최적화 미수행
   - 향후: 교차검증으로 최적 임계값 탐색

5. **세부 타입 미검증**
   - selftest.py는 4시즌만 검증, light/bright/soft 등 세부 타입 미검증
   - 향후: 세부 타입별 회귀 테스트 추가

### 배포 제약

1. **화이트밸런스 미적용 학습**
   - 원본 색에서 라벨 추출 (조명 색온도 편향 있을 수 있음)
   - 향후: gray_world_white_balance 전처리 옵션

2. **SegFormer 세그멘테이션 모델 로드 비용**
   - method=seg 사용 시 첫 실행에서 HuggingFace 허브에서 자동 캐시
   - 향후: 모바일/경량 세그멘테이션 모델 대체 (예: SINet)

3. **색조화도 임계값 문화 최적화 부재**
   - 현재: ΔE<12(좋음), <28(무난) - 일반적 지각 임계값 참고
   - 미최적화: 한국 패션 문화 기준으로 조정 미수행
   - 향후: A/B 테스트로 임계값 재설정

## 회고: 기술과 실무의 격차

### 처음 예상

프로젝트 시작 당시, 저는 색상 추천을 단순한 규칙 기반 시스템으로 생각했습니다. 옷 이미지에서 색을 뽑고, 보색이나 유사색을 규칙으로 찾으면 된다고 봤습니다. 하지만 실제로는 다음과 같은 깊이 있는 문제들을 마주했습니다.

### 마주친 기술적 난제

**1. 색공간 문제:**
RGB 좌표계에서 단순히 거리를 계산하면, (255,0,0) 빨강과 (0,255,0) 초록 사이의 거리가 (255,0,0)과 (200,0,0) 진한 빨강 사이의 거리와 같이 계산됩니다. 하지만 인간의 눈으로는 후자가 훨씬 비슷합니다. 이는 RGB 색공간이 지각적으로 비균일하기 때문입니다. CIELab으로 변환하고 ΔE76 거리를 사용하면서 이 문제를 해결했고, 이를 Python과 TypeScript 양쪽에 일관되게 적용했습니다.

**2. 의사라벨 품질:**
처음엔 단일 규칙을 FairFace 코퍼스 전체에 적용했는데, 인구통계별 피부색 편차가 예상보다 크더군요. 특히 동양인 피부의 황색도(Lab b*)와 밝기(L*)가 유럽인과 상당히 달랐습니다. 결국 2-pass 부트스트랩 라벨링을 설계했습니다. 첫 번째는 다양한 피부색을 확보하고, 두 번째는 코퍼스의 중앙값으로 규칙의 중립점을 재설정하는 방식입니다. 이것이 CNN 모델의 성능 상한을 좌우한다는 것을 깨달았습니다.

**3. 학습-추론 분포 일치:**
모델을 학습할 때 224×224 정사각 얼굴 이미지를 사용했는데, 추론 단계에서도 정확히 같은 방식으로 크롭해야 합니다. mediapipe로 얼굴 bbox를 잡아 1.25배 정사각 크롭 후 224 리사이즈하는 로직을 predict.py와 personal_color.py(서버)에서 동일하게 구현했습니다. 이 부분을 간과하면 추론 정확도가 급락합니다.

**4. 팀 병렬 개발 인터페이스:**
프로젝트 초기에는 각자 코드를 작성했다가 통합할 때마다 충돌이 발생했습니다. API 응답 구조가 다르거나, 카테고리명이 불일치하는 식의 문제였습니다. 해결책은 사전 설계서였습니다. MVP 범위, 모듈 분해, API 스펙, 데이터 포맷을 코드 작성 전에 합의함으로써 훨씬 수월하게 진행할 수 있었습니다.

### 배운 점

이 경험을 통해 저는 다음을 깨달았습니다:

1. **색공간 선택이 핵심:** 수학적 기초가 없으면 나중에 돌이킬 수 없습니다.
2. **의사라벨 품질이 모델 성능을 결정:** "쓰레기 입 = 쓰레기 출(garbage in, garbage out)" - 규칙 설계와 코퍼스 보정의 중요성을 체감했습니다.
3. **분포 일치의 중요성:** 학습과 추론의 환경이 다르면 성능이 급락합니다.
4. **문서화와 인터페이스 정의:** 기술적 우수성만큼이나 팀 협업을 위한 명확한 스펙이 중요합니다.

### 향후 전망

현재 시스템은 얼굴 사진으로 퍼스널컬러를 판정하고, 시즌 팔레트를 기준으로 옷을 추천합니다. 향후는 다음 방향으로 발전시킬 계획입니다:

1. 사용자의 실제 의류 데이터베이스와 연동
2. 계절 변화나 트렌드를 반영한 추천
3. STRONG=0.6 같은 임계값을 베이지안 최적화로 조정
4. 다양한 인구통계 데이터셋에서의 성능 검증
5. 웹사이트나 앱으로 서비스 배포

이 프로젝트는 패션 의류 업계에서 7년간 쌓은 상품기획 경험과 AI 기술을 처음으로 의미 있게 연결한 경험입니다. 강의실 이론이 아닌 실제 코드에서 색공간 변환의 수학이, 신뢰도 지표의 설계가, 팀 협업의 필요성이 검증될 때의 배움은 깊고 오래 남을 것 같습니다.
