## 1. 문서 목적

본 문서는 김도윤이 전담하는 **2D 색상 추천 시스템**의 범위·구조·인터페이스·일정·품질 목표를 코드 작성 이전에 확정하기 위한 상세 설계서이다.
팀원(박찬영-옷 착용, 오필립-체형 3D, 이은석-Vision/통합)과의 **연동 지점**을 명확히 하여 병렬 개발이 가능하도록 한다.

## 2. 담당 범위 (Scope)

### 2.1 MVP 한 문장 정의
> "마네킹에 입힌 **옷 이미지에서 대표 색을 추출**하고, **색상 조화 이론에 따라 어울리는 다른 카테고리 옷의 색을 추천**하여 코디를 제안하는 시스템."

### 2.2 In-Scope (담당)
- 옷 이미지 → 대표 색 팔레트 추출 (색상 추출 모듈)
- 추출 색 → 어울리는 색 조합 추천 (추천 엔진)
- (선택) 웜톤/쿨톤 퍼스널 컬러 분류
- 추천 결과 제공 API (FastAPI 엔드포인트)
- 프론트 연동용 추천 결과 표시 컴포넌트 (협업)

### 2.3 협업 / 연관 (Shared)
- **AI 옷 사진 → 2D Combine (IDM-VTON 합성)**: 이은석(Vision)과 공동 PoC
- 추천 결과의 3D 반영: 박찬영(의류 부착) 파트와 카테고리/색 매핑 합의

### 2.4 Out-of-Scope (비담당)
- 3D 렌더링, 본 리깅, 체형 슬라이더 (오필립/박찬영)
- TripoSR/Wonder3D 3D 복원 (이은석 외)

## 3. 시스템 아키텍처

### 3.1 모듈 분해

| 모듈 | 입력 | 처리 | 출력 |
|------|------|------|------|
| **① 색상 추출** | 옷 이미지(.png/.jpg) | 배경 제거 → LAB 색공간 KMeans 클러스터링 | 대표 색 팔레트 N개 (hex + 비율) |
| **② 추천 엔진** | 대표 색(hex) + 대상 카테고리 | HSV Hue 기반 색상 조화 규칙 적용 | 추천 색 리스트 (hex + 조화유형 + 사유) |
| **③ API/연동** | 모듈①②를 HTTP로 노출 | FastAPI 라우팅 | JSON 응답 |

### 3.2 데이터 흐름

```
[옷 이미지/썸네일]
      │  (박찬영·이은석이 제공하는 의류 자산)
      ▼
┌─────────────────┐
│ ① 색상 추출      │  배경제거 → LAB KMeans
└─────────────────┘
      │  palette = [#A1B2C3, ...]
      ▼
┌─────────────────┐
│ ② 추천 엔진      │  보색/유사색/삼각배색 + (웜/쿨)
└─────────────────┘
      │  recommendations = [{category, hex, reason}]
      ▼
┌─────────────────┐
│ ③ FastAPI       │  POST /recommend
└─────────────────┘
      │  JSON
      ▼
[Next.js 프론트]  팔레트 칩 + 추천 썸네일 표시
```

## 4. 모듈별 상세 설계

### 4.1 모듈 ① — 색상 추출
- **배경 제거:** `rembg`(U2-Net) 또는 의류 자산이 알파 채널/마스크를 가질 경우 그대로 활용
- **색공간:** RGB → **CIE LAB** 변환 후 클러스터링
  - 이유: LAB는 지각적 균일성(perceptual uniformity)이 높아 사람 눈의 색 차이와 일치. RGB는 색 거리 왜곡이 큼.
- **클러스터링:** `scikit-learn` KMeans, k=3~5
- **후처리:** 클러스터별 픽셀 비율 계산 → 비율순 정렬, 무채색(저채도) 필터 옵션
- **산출:** `[{hex, ratio}]`

### 4.2 모듈 ② — 추천 엔진 (차별화 포인트)
- 대표 색 → **HSV의 Hue(0~360°)** 변환
- 색상 조화 규칙으로 추천색 생성:
  | 조화 유형 | Hue 변환 | 느낌 |
  |-----------|----------|------|
  | 보색(Complementary) | +180° | 강한 대비, 포인트 |
  | 유사색(Analogous) | ±30° | 차분, 통일감 |
  | 삼각배색(Triadic) | ±120° | 활기, 균형 |
  | 분할보색(Split-Comp.) | +150°/+210° | 안정적 대비 |
- (선택) **퍼스널 컬러 필터:** 추출 색의 웜/쿨 톤 판별 후 추천 색 한 겹 필터링 → 발표 어필
- **출력에 `reason` 필드 포함:** "상의 #2E4C8A(쿨톤 블루)의 유사색으로 통일감 있는 코디" → 발표 설득력 확보
- **라이브러리:** `colour-science` / `colormath`, 또는 색공간 변환 직접 구현

### 4.3 모듈 ③ — API / 연동
- **프레임워크:** FastAPI (계획서의 "AI 추론 게이트웨이" 구조와 일치)
- **엔드포인트:** `POST /recommend`
- **프론트:** Next.js에서 JSON 수신 → 팔레트 칩 + 추천 색 썸네일 렌더링

## 5. 인터페이스 정의 (팀 연동 계약)

### 5.1 추천 API 스펙

**요청** `POST /recommend`
```json
{
  "image_url": "https://.../top_001.png",   // 또는 base_color
  "base_color": "#2E4C8A",                   // image_url 없을 때
  "target_category": "bottom",               // top|bottom|outer|shoes
  "harmony": ["analogous", "complementary"]  // 원하는 조화 유형
}
```

**응답**
```json
{
  "palette": [
    {"hex": "#2E4C8A", "ratio": 0.62},
    {"hex": "#D9DEE8", "ratio": 0.21}
  ],
  "tone": "cool",
  "recommendations": [
    {"category": "bottom", "hex": "#8A6B2E", "harmony": "complementary",
     "reason": "상의 쿨톤 블루의 보색으로 포인트를 주는 조합"}
  ]
}
```

### 5.2 연동 포인트 (누구와 무엇을 합의하나)
| 상대 | 합의 항목 |
|------|-----------|
| 박찬영(의류 부착) | 카테고리 명칭(top/bottom/outer/shoes), 의류 썸네일 경로 규칙 |
| 이은석(통합/Vision) | API 호출 방식, IDM-VTON 2D 합성 협업 범위 |
| 오필립(체형) | (간접) 추천 결과의 마네킹 표시 위치 |

## 8. 품질 목표 / 테스트

| 지표 | 목표값 |
|------|--------|
| 팔레트 정확도 | 추출 색이 육안상 옷 실제 색과 일치 (주관 평가) |
| 추천 API 응답시간 | ≤ 200 ms (로컬 이미지, base_color 입력 시) |
| 추천 설명 가능성 | 모든 추천에 `reason` 제공 (색채 이론 근거) |
| 카테고리 커버리지 | top/bottom/outer/shoes 4종 모두 추천 가능 |

**테스트 방법:** 샘플 옷 이미지 10~20장에 대해 팔레트·추천 결과를 표로 정리하여 육안 검증.
